# API Key 管理（管理端）

## 功能简介

API Key 管理是 LLM 网关的**管理端令牌管理**功能，允许系统管理员查看、创建和管理平台中所有的 API Key。API Key 是调用 LLM 网关 API 的认证凭证——每个 API 请求必须携带有效的 API Key 才能被网关接受和处理。

与用户在 Console 自助管理的个人 Token 不同，管理端的 API Key 管理提供全局视角，管理员可以查看所有用户的 Key，并拥有更多的管控能力（如 IP 白名单、细粒度限流等）。

> 💡 提示: 用户在 Console 的 [API Token](../../console/chatapp/token.md) 页面可以自助创建和管理自己的 Token。管理端 API Key 管理则面向系统管理员，提供全局的 Key 管控视角。

## 进入路径

BOSS → LLM 网关 → **API Key**

路径：`/boss/tokens`

## API Key 列表

![API Key 列表](/screenshots/boss/gateway-apikeys-list.png)

| 列 | 字段 | 说明 | 备注 |
|----|------|------|------|
| 名称 | `name` | Key 的自定义名称 | 用于标识 Key 用途 |
| API Key | `apiKey` | Key 值（部分隐藏） | 以 `sk-xxxx...xxxx` 格式展示，支持一键复制到剪贴板 |
| 归属 | `account` / `belongTo` | Key 所属的用户/账户 | 显示创建者信息 |
| RPM 限制 | `rateLimitRPM` | 每分钟请求数限制 | 为 0 时显示 `∞`（无限制） |
| TPM 限制 | `rateLimitTPM` | 每分钟 Token 数限制 | 以 K 为单位显示（如 `100K`） |
| 允许 IP | `allowedIPs` | IP 访问白名单 | 使用 `CollapseItem` 折叠展示，无限制时显示 `*` |
| 过期时间 | `expiresAt` | Key 的过期时间 | 已过期的 Key 显示 `expired` 标签（Chip） |
| 创建时间 | `createdAt` | Key 的创建时间 | 时间戳格式 |
| 操作 | — | 编辑 / 删除 | — |

### Key 掩码显示

API Key 在列表中以掩码形式展示，仅显示前缀和后缀：

```
sk-abc1...xyz9
```

点击 Key 旁的复制按钮可将完整 Key 复制到剪贴板。

> ⚠️ 注意: API Key 的完整值仅在**创建时**显示一次。创建后，系统仅存储 Key 的哈希值，无法再次查看完整 Key。请在创建时立即复制并妥善保管。

### Key 状态

| 状态 | 说明 | 列表表现 |
|------|------|---------|
| `active` | 活跃状态，可正常使用 | 正常显示 |
| `expired` | 已超过过期时间 | `expiresAt` 列显示红色 `expired` Chip 标签 |

> 💡 提示: 已过期的 Key 不会被自动删除，但无法再用于 API 认证。管理员可以手动删除过期 Key 或编辑其过期时间延长有效期。

## 创建 API Key

点击 **创建** 按钮打开创建表单：

![创建 API Key](/screenshots/boss/gateway-apikey-create.png)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名称 | 文本 | ✅ | Key 的自定义名称，用于标识用途 |
| 归属用户 | 用户选择器 | ✅ | 指定 Key 所属的用户账户 |
| RPM 限制 | 数字 | — | 每分钟最大请求数，0 表示不限制 |
| TPM 限制 | 数字 | — | 每分钟最大 Token 数，0 表示不限制 |
| 允许 IP | IP/CIDR 列表 | — | IP 访问白名单，留空表示不限制 |
| 过期时间 | 日期时间选择器 | — | Key 的过期日期，留空表示永不过期 |

### 创建成功

创建成功后，系统会弹出对话框展示完整的 API Key：

```
✅ API Key 创建成功

您的 API Key：
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

⚠️ 请立即复制此 Key，关闭对话框后将无法再次查看完整值。
```

> ⚠️ 注意: 这是查看完整 API Key 的唯一机会。关闭对话框后，系统只保留 Key 的哈希值用于验证，无法恢复原始 Key。

## 速率限制

API Key 支持两个维度的独立限流：

### RPM（Requests Per Minute）

每分钟允许的最大请求次数。超限后返回 HTTP `429 Too Many Requests`。

| 设置 | 效果 |
|------|------|
| `0` | 不限制（列表显示 `∞`） |
| `60` | 每分钟最多 60 次请求（每秒约 1 次） |
| `600` | 每分钟最多 600 次请求（每秒约 10 次） |

### TPM（Tokens Per Minute）

每分钟允许消耗的最大 Token 数量。列表中以 K 为单位显示（如 `100K` = 100,000 Tokens/分钟）。

| 设置 | 效果 |
|------|------|
| `0` | 不限制（列表显示 `∞`） |
| `100000` | 每分钟最多 10 万 Token（列表显示 `100K`） |
| `1000000` | 每分钟最多 100 万 Token（列表显示 `1000K`） |

> 💡 提示: RPM 和 TPM 是独立计算的，任一维度超限都会触发限流。对于高频短请求场景侧重 RPM 限制；对于低频长文本场景侧重 TPM 限制。

## IP 访问白名单

每个 API Key 可以配置独立的 IP 访问白名单，限制只有特定 IP 地址才能使用该 Key。

- 支持单个 IP 地址（如 `192.168.1.100`）和 CIDR 网段（如 `10.0.0.0/8`）
- 列表中使用 `CollapseItem` 组件折叠展示多个 IP
- 未配置白名单时显示 `*`（允许所有 IP）

> 💡 提示: IP 白名单与 [网关配置](./config.md) 中的全局白名单是独立的两层过滤。请求需同时满足两层白名单才能通过。

## 管理端 vs 用户自助 Token

| 对比项 | 管理端 API Key | 用户自助 Token |
|--------|---------------|---------------|
| 管理入口 | BOSS → LLM 网关 → API Key | Console → ChatApp → API Token |
| 操作权限 | 系统管理员 | 普通用户（管理自己的 Token） |
| 可见范围 | 全平台所有 Key | 仅自己的 Token |
| IP 白名单 | ✅ 支持 | — |
| RPM/TPM 限制 | ✅ 支持 | ✅ 支持 |
| 查看归属用户 | ✅ 可见 | — |

## 编辑 API Key

点击列表中的 **编辑** 按钮，可修改以下字段：

- 名称
- RPM / TPM 限制
- 允许 IP 列表
- 过期时间

> 💡 提示: API Key 的值本身无法修改。如需更换 Key 值，需删除后重新创建。

## 删除 API Key

点击 **删除** 按钮，确认弹窗后删除 Key。删除后使用该 Key 的所有 API 请求将立即失败。

> ⚠️ 注意: 删除操作不可撤销。删除前请确认没有正在使用该 Key 的生产服务，否则会导致服务中断。

## 安全建议

### Key 生命周期管理

1. **定期轮换**：建议生产环境的 API Key 每 90 天轮换一次
2. **最小权限**：为不同用途创建独立的 Key，避免共享
3. **及时清理**：已过期或不再使用的 Key 应及时删除
4. **IP 白名单**：生产环境的 Key 务必配置 IP 白名单
5. **监控告警**：通过 [运营概览](./operations.md) 监控 Key 的使用情况，发现异常及时处理

### Key 泄露处理

如发现 API Key 泄露：

1. 立即在管理端删除该 Key
2. 为相关服务创建新的 Key
3. 检查 [审计日志](./audit.md) 确认是否有异常调用
4. 通知相关用户更新 Key 配置

## 常见问题

### 创建 Key 后忘记复制怎么办？

无法恢复，只能删除后重新创建新的 Key。

### RPM 和 TPM 可以只设置其中一个吗？

可以。设置为 `0` 的维度表示不限制。例如只设 RPM=60、TPM=0 表示限制请求频率但不限制 Token 用量。

### API Key 过期后还能自动续期吗？

不能自动续期。需要管理员手动编辑 Key 设置新的过期时间，或删除后创建新 Key。

## 权限要求

需要 **系统管理员** 角色。系统管理员可以查看和管理平台中所有用户的 API Key。
